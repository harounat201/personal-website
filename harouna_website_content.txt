[2] Amazon
Role: Software Development Engineer Intern
Team: Devices — Fire Tablet Launcher
Dates: May 2025 – August 2025
Location: Seattle, WA

PROJECT DEEP DIVE — Alexa Utterance Validation System:
Problem:
Next-gen Fire Tablets needed an accurate, up-to-date data bank of potential on-device Alexa actions, reflective of real Tablet capabilities. The existing JSON metadata of Alexa capabilities was unvalidated and too verbose for reliable LLM prompt injection at runtime.

Materials & Tools:
- Raw JSON data of Alexa capabilities
- Internal Alexa Utterance Validation Framework (TTS-based)
- AWS Step Functions, AWS Bedrock (Claude Sonnet), AWS Lambda, SNS, S3

Solution — Serverless Cloud Workflow:
Built a multi-stage serverless pipeline using AWS Step Functions with the following flow:

STAGE 1 — Initial Description Generation:
- Lambda queries and filters raw JSON data from S3
- Distributed Processing (Map) State fans out across JSON items in parallel
- Claude Sonnet (via Bedrock) summarizes each JSON capability into natural language utterances
- Output prefix mapped and redirected to S3 keys; generated utterances CSV stored in S3

STAGE 2 — Utterance Validation Framework:
- Validation Service receives generated utterances
- SNS Topic dispatches validation tasks
- Workflow Gateway (Wait for Callback) uses callback token mechanism to pause execution
- SNS Topic Listener Lambda validates SNS responses and returns callback token + batch ID
- Utterance validation results stored in S3

STAGE 3 — Final Description Generation:
- Distributed Processing (Map) State fans out again
- Claude Sonnet evaluates each utterance against validation results
- Lambda evaluates utterances; results stored as Utterance Ranges in S3
- Final JSON output written to JSON Output Data S3 bucket
- EventBridge triggers StepFunctions Workflow Definition for orchestration

Result:
A clean, validated store of accurate on-device Alexa capabilities — concise and structured for LLM prompt injection during Fire Tablet runtime. Reduced JSON processing time by 87% via parallel Map state distribution.

Resume Bullets:

- Designed a callback token mechanism with TypeScript SNS listeners and Lambda gateways to synchronize validation responses
